# Fix 404 Error - Vercel Configuration

## The Problem
Vercel is returning 404 because it can't find your `index.html` file.

## Solution: Configure Output Directory in Vercel Dashboard

### Step 1: Go to Vercel Project Settings
1. Open [vercel.com](https://vercel.com)
2. Go to your project: **maldito-sur-page**
3. Click **"Settings"** (top navigation)

### Step 2: Update Output Directory
1. In Settings, click **"General"** (left sidebar)
2. Scroll down to **"Build & Development Settings"**
3. Find **"Output Directory"** field
4. **Change it to: `.`** (a single dot) or **leave it EMPTY**
5. Click **"Save"**

### Step 3: Update Framework Preset
1. In the same **"General"** settings page
2. Find **"Framework Preset"**
3. Set it to **"Other"** or **"Vite"**
4. Or leave it as **"Other"**

### Step 4: Redeploy
1. Go to **"Deployments"** tab
2. Click the three dots (⋯) on your latest deployment
3. Click **"Redeploy"**
4. Wait for it to complete

### Step 5: Verify
Visit: `https://maldito-sur-page.vercel.app/`
Should now work! ✅

## Alternative: If Settings Don't Work

If you can't change the Output Directory setting, create a `public` folder:

```bash
mkdir public
move index.html public\index.html
move image.jpg public\image.jpg
```

Then update Vercel settings:
- **Output Directory**: `public`

But the first solution (Output Directory = `.`) should work!

## Current File Structure (Correct)

Your files should be:
```
your-project/
├── api/
│   └── schedule.js
├── data/
│   └── schedule.json
├── index.html      ← Should be in root
├── image.jpg
├── package.json
└── (no vercel.json needed)
```

After setting Output Directory to `.`, Vercel will serve `index.html` from root.
